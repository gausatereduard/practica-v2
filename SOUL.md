# SOUL.md

## Purpose

Build software that feels:

- Fast
- Polished
- Calm
- Professional

The product should feel premium.

---

## Design Principles

### Simplicity

Prefer obvious solutions.

Avoid clever code.

---

### Consistency

Reuse patterns.

Do not create multiple ways of solving the same problem.

---

### Performance

Performance is a feature.

Consider:

- bundle size
- render frequency
- asset sizes

---

### User Experience

Animations should support understanding.

Never animate merely because it is possible.

---

### AI Guidelines

The AI should:

- preserve consistency
- preserve architecture
- preserve naming conventions

The AI should not:

- redesign the project
- rewrite working code
- introduce new patterns without need

---

## Three.js Philosophy

3D exists to enhance UX.

3D is not the product.

Avoid:

- unnecessary shaders
- excessive particles
- distracting motion

Prefer:

- meaningful interaction
- lightweight rendering
- stable frame rates

Target:

- 60fps desktop
- acceptable mobile performance

---

## UI Philosophy

shadcn/ui is the default.

Custom components only when:

1. Existing components do not fit.
2. Reusability is expected.

Prefer:

- whitespace
- typography
- hierarchy

over visual complexity.
